File Structure

  • dist - All files are here compiled from src folder using Node.js / Gulp . dist files are ready to edit as you intend or upload live to the server. Inside dist folder you'll find:
    • assets - folder contains all necessary files except the HTML files. Here are the files below:
      • css - contains all css files necessary for the template style.
      • fonts - contains self hosted fonts for Bootstrap icons.
      • img - contains all images for the template.
      • js - contains all script files used in the template.
    • All HTML files
  • src - Contains all template source files like HTML / SCSS / CSS / JS that are then processed (compiled) to dist folder. Inside src folder you'll find:
    • assets - folder contains all necessary files except the HTML files. Here are the files below:
      • src/assets/fonts - contains self hosted fonts for Bootstrap icons.
      • src/assets/img - contains all images for the template.
      • src/assets/js - contains main app.js and JS vendor files in vendor folder. If you don't wish to use all included plugins and scripts, just remove unwanted vendor JS files from src/assets/js/vendor then remove unwanted functions from src/assets/js/app.js and recompile.
      • src/assets/scss/vendor - Contains CSS vendor files. If you don't wish to use all included plugins, just remove unwanted vendor CSS from src/assets/scss/vendor and recompile.
      • src/assets/scss - Contains all project SCSS files that are compiled and minified in dist/assets/css folder.
    • layouts - is a common HTML layout that every page shares.
    • pages - These files will just have the middle section of the design, since the layout already covers the top and bottom
    • partials - Partials are a feature of Handlebars which allow you to inject HTML anywhere in a page or layout.
  • package.json - Includes the list of dependencies to install from npm.
  • gulpfile.json - Configuration file for Gulp library. It contains all tasks to perform with Gulp. Learn more about it from official Gulp Documentation

Working With Template

There are two ways to work with template.

  1. Direct Editing
  2. Adnvanced Editing

Direct Editing

For direct editing and less hassle just go to dist folder and edit HTML, CSS and Scripts or changes image as you required and your template is ready to live.

Adnvanced Editing

This step aims for advanced users who want to speed up development process and is optional. You can still use plain HTML / CSS / JS to customize template. Files you need are located in dist

If you'd like to speed up development process with Node.js / Gulp / SASS , then follow these steps:

  • Install Node.js if you don't have it yet.
  • Run npm install gulp-cli -g command, if you haven't already installed Gulp CLI previously.
  • Unzip the template package file named main and in the root project folder you'll get documentation & nextai , go to your command line tool and change directory to the nextai folder and run npm install OR you can open nextai folder in vscode editor and open terminal from vscode editor, now run npm install . This will install the npm packages listed in the package.json file.
  • You should now have the project files set up and all the npm packages installed.
After running npm install you'll see some warning messages in terminal. Don't worry, this is due to incompatibility between NodeJS versions, it does not affect your development process, please ignore the warning messages.

Gulp Commands

  • npm run build This command will fire default gulp task which includes: creating dist folder, compiling and merging vendor libraries, JS task, SCSS and HTML. You have to run this command once at the begining of project.
  • npm run start This command will launch watch task and BrowserSync. It will watch changes of your HTML / SCSS / CSS / JS files and automatically compile them.
  • When you want to kill the server just hit Control + C

Quick Installation

  • Installing npm packages (Terminal command: npm install )
  • Compiling and merging vendor libraries, JS task, SASS and HTML (Terminal command at the begining of the project once: npm run build )
  • Running gulp (Terminal command: npm start )
